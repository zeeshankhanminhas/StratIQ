# Strat IQ — Responsive Editorial Design Brief

## Objective

Create a premium landing page for **Strat IQ**, a Business Intelligence & Growth Advisory, designed at a 1440px desktop width and fully responsive across tablet and mobile.

The visual direction is inspired by the overall design language, visual rhythm, spacing system, typography scale, section proportions, and editorial layout philosophy of the Clarify Webflow template.

This is not permission to reproduce the reference template's branding, content, illustrations, imagery, icons, wording, or proprietary assets. The resulting site must remain unmistakably original to Strat IQ.

## Brand impression

The site should communicate:

- Executive confidence
- Structured thinking
- Clarity
- Intelligence
- Evidence-based decision making
- Premium consulting

The overall feeling should be: **A world-class intelligence firm with exceptional taste.**

## Information architecture

The page should progress as a continuous editorial story:

1. Large hero
2. Editorial statement
3. Alternating content sections
4. Capabilities
5. Approach
6. Proof
7. Insights
8. Final call to action

Each section should feel like a new chapter rather than another component block.

## Section proportions

- Major desktop sections should generally occupy 80–120vh when content supports it.
- Use generous vertical spacing rather than compressing content.
- Whitespace must be treated as a functional part of hierarchy and pacing.
- Tablet and mobile sections may use natural content height, while retaining strong pauses between chapters.

## Layout rhythm

Alternate composition patterns throughout the page:

- Text → visual
- Visual → text
- Large statement
- Split editorial layout
- Wide visual
- Numbered section
- Editorial list

Do not repeat the same heading-plus-card-grid pattern across adjacent sections.

## Typography

- Use oversized editorial serif headlines.
- Use clean modern sans-serif body copy.
- Keep display headlines tightly tracked with controlled line height.
- Use typography as the primary hierarchy mechanism.
- Avoid excessive decorative UI elements.
- Scale type fluidly with `clamp()` across breakpoints.

## Grid

- 12-column desktop grid.
- Maximum content width: 1280–1320px.
- Consistent outer margins and column alignment.
- Tablet layouts should typically collapse to 6–8 conceptual columns.
- Mobile layouts should become a clear single-column editorial flow.

## Spacing system

Primary spacing values:

- 32px
- 48px
- 64px
- 96px
- 160px

Use the largest values between major sections and smaller values within tightly related content groups.

## Colour system

- Warm Ivory — primary background
- Soft Stone — secondary surfaces
- Midnight Navy — dark sections
- Muted Bronze — accent
- Near Black — primary typography
- Slate Grey — supporting typography

Avoid bright colours, neon, heavy gradients, and glassmorphism.

## Components

### Navigation

- Thin, editorial, and generously spaced.
- Transparent over the hero.
- Add subtle blur and border after scrolling.
- Collapse to a clear mobile menu.

### Buttons

- Minimal and restrained.
- Soft radius.
- Clear hover and focus states.
- Avoid oversized pill treatments unless compositionally justified.

### Cards

- Use sparingly.
- Prefer large editorial surfaces over repeated feature cards.
- Use hairline borders and minimal shadow.
- Avoid dashboard widgets and generic SaaS feature grids.

## Visual language

Use original abstract intelligence visuals such as:

- Structured compositions
- Documents and decision briefs
- Knowledge objects
- Connection lines
- Editorial diagrams
- Information layers
- Research artefacts

Do not use stock photography, office scenes, people, handshakes, laptops, generic analytics charts, or generic marketing illustrations.

## Motion

- Navigation blur on scroll
- Fade-up section reveals
- Gentle visual scaling
- Soft hover elevation
- Slow, controlled transitions
- Respect `prefers-reduced-motion`

Avoid bouncing, aggressive parallax, and exaggerated animation.

## Responsive behaviour

### Desktop

- Preserve full editorial scale and asymmetry.
- Use large visual fields and generous whitespace.
- Keep section layouts deliberately varied.

### Tablet

- Reduce headline size without losing drama.
- Collapse three-column layouts to two or one where appropriate.
- Preserve asymmetry through spacing and ordering rather than cramped columns.
- Keep touch targets at least 44px.

### Mobile

- Use a single-column flow.
- Stack visual and copy blocks in narrative order.
- Reduce section spacing proportionally, but retain clear chapter breaks.
- Avoid absolute-positioned content that causes overlap.
- Convert large visual compositions into contained, readable mobile arrangements.
- Keep body text at a comfortable reading size and line length.

## Non-copying boundary

Do not reproduce the reference template's:

- Wording
- Headlines
- Copy
- Icons
- Illustrations
- Imagery
- Graphics
- Branding
- Logos
- Exact colours
- Proprietary assets

Only draw inspiration from its structural hierarchy, layout rhythm, section proportions, spacing, typography scale, editorial composition, visual pacing, and overall experience philosophy.

The final design must be original to Strat IQ while reaching the same level of sophistication, clarity, and editorial elegance.
